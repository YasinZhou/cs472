1. Can post requests be bookmarked? What are the problems?
POST request shouldn't be bookmarked. Because a bookmark only consists of the URL, but all of the form parameters in post request are not in the URL, they will be lost. The web server won't know what to do with the request without expecting form parameters.

2. What is the purpose of request dispatching?
The purpose of request dispatching is to forward a client's request to a resource or include the resource itself in the response back to the client. A resource can be another servlet, or an HTML file, or a JSP file, etc.

3. What is the difference between redirect and request dispatch?
Redirect: It will jump to a new url. It goes to a different server file. It only sends message, won't send request and response objects. It will create new request and response objects.
Request dispatch: It will jump to a new servlet. It goes to the same server file. It will send request and response objects to destination servlet.Browser is completely unaware of servlet forward and hence the URL in browser address bar will remain unchanged.

4. What is an attribute?
It is an object bound into one of the three servlet API objects on the web server: HttpServletRequest, HttpSession, and ServletContext. It has a name-value pair( name is String, value has type Object). They promote communication between components.

5. What is the difference between attributes and parameters?
1: Parameters are read only, attributes are read/write objects.
2: Parameters are String objects, attributes can be objects of any type.

6. What are dangers of using attributes?
Only request scope attribute is thread-safe. Session scope attribute and Context scope attribute is not thread-safe.

9. How long does a session last?
After a session is created, the container can’t tell when browser is finished with session. So we remove session by 3 ways: use session-timeout by days, use session.setMaxInactiveInterval(20*60) by seconds, or use session.invalidate() to remove it immediately.

10. What is a cookie, and how long does a cookie last?
1. A cookie is a text file stored on the client computer that is persisted between the multiple client requests. Session tracking via cookie exchange
2. Temporary cookie: browser removes when it closes. Permanent cookie: has a max age set.
